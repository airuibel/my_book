    \section{Docker介绍}\label{dockerux4ecbux7ecd}

\begin{verbatim}
Docker中文社区文档
Docker 是一个开源的软件部署解决方案。
Docker 也是轻量级的应用容器框架。
Docker 可以打包、发布、运行任何的应用。
Docker 就像一个盒子，里面可以装很多物件，如果需要某些物件，可以直接将该盒子拿走，而不需要从该盒子中一件一件的取。
Docker 是一个客户端-服务端(C/S)架构程序。
客户端只需要向服务端发出请求，服务端处理完请求后会返回结果。
Docker 包括三个基本概念:
\end{verbatim}

\paragraph{镜像（Image）}\label{ux955cux50cfimage}

\begin{itemize}
\tightlist
\item
  Docker的镜像概念类似于虚拟机里的镜像，是一个只读的模板，一个独立的文件系统，包括运行容器所需的数据，可以用来创建新的容器。
  例如：一个镜像可以包含一个完整的 ubuntu
  操作系统环境，里面仅安装了MySQL或用户需要的其它应用程序。
\end{itemize}

\paragraph{容器（Container）}\label{ux5bb9ux5668container}

\begin{itemize}
\tightlist
\item
  Docker容器是由Docker镜像创建的运行实例，类似VM虚拟机，支持启动，停止，删除等。
  每个容器间是相互隔离的，容器中会运行特定的应用，包含特定应用的代码及所需的依赖文件。
\end{itemize}

\paragraph{仓库（Repository）}\label{ux4ed3ux5e93repository}

\begin{itemize}
\tightlist
\item
  Docker的仓库功能类似于Github，是用于托管镜像的。
\end{itemize}

\paragraph{dock中文社区}\label{dockux4e2dux6587ux793eux533a}

http://www.docker.org.cn/index.html

    \subsection{1.Docker镜像操作}\label{dockerux955cux50cfux64cdux4f5c}

\begin{itemize}
\tightlist
\item
  查看所有镜像列表
\end{itemize}

\begin{verbatim}
sudo docker image ls

或者

sudo docker images
\end{verbatim}

\begin{itemize}
\tightlist
\item
  REPOSITORY：镜像所在的仓库名称
\item
  TAG：镜像标签
\item
  IMAGEID：镜像ID
\item
  CREATED：镜像的创建日期(不是获取该镜像的日期)
\item
  SIZE：镜像大小
\end{itemize}

\subsection{2.安装镜像}\label{ux5b89ux88c5ux955cux50cf}

\begin{itemize}
\tightlist
\item
  2.1 从仓库拉取镜像
\end{itemize}

\begin{verbatim}
官方镜像
sudo docker image pull 镜像名称 或者 sudo docker image pull library/镜像名称
sudo docker image pull ubuntu 或者 sudo docker image pull library/ubuntu
sudo docker image pull ubuntu:16.04 或者 sudo docker image pull library/ubuntu:16.04
\end{verbatim}

\begin{verbatim}
个人镜像
sudo docker image pull 仓库名称/镜像名称

\end{verbatim}

\begin{itemize}

    \section{Docker容器操作}\label{dockerux5bb9ux5668ux64cdux4f5c}

\begin{itemize}
\tightlist
\item
  查看正在运行的容器
\end{itemize}

\begin{verbatim}
sudo docker container ls

#查看所有的容器
sudo docker container ls --all
\end{verbatim}

\begin{itemize}
\tightlist
\item
  2.创建新容器
\end{itemize}

\begin{verbatim}
sudo docker run [option] 镜像名 [向启动容器中传入的命令]
\end{verbatim}

\begin{itemize}
\item
  常用可选参数说明：

\begin{verbatim}
* -i 表示以《交互模式》运行容器。
* -t 表示容器启动后会进入其命令行。加入这两个参数后，容器创建就能登录进去。即分配一个伪终端。
* --name 为创建的容器命名。
* -v 表示目录映射关系，即宿主机目录:容器中目录。注意:最好做目录映射，在宿主机上做修改，然后共享到容器上。 
* -d 会创建一个守护式容器在后台运行(这样创建容器后不会自动登录容器)。 
* -p 表示端口映射，即宿主机端口:容器中端口。
* --network=host 表示将主机的网络环境映射到容器中，使容器的网络与主机相同。
\end{verbatim}
\item
  3.交互式容器
\end{itemize}

\begin{verbatim}
sudo docker run -it --name=ubuntu1 ubuntu /bin/bash
\end{verbatim}

\begin{itemize}
\item
  **在容器中可以随意执行linux命令，就是一个ubuntu的环境。
\item
  \textbf{当执行 exit 命令退出时，该容器随之停止。}
\item
  4.开启守护式容器
\end{itemize}

\begin{verbatim}
sudo docker run -dit --name=ubuntu2 ubuntu
\end{verbatim}

\begin{itemize}
\tightlist
\item
  进入到容器内部交互环境
\end{itemize}

\begin{verbatim}
sudo docker exec -it 

容器名或容器id 进入后执行的第一个命令

sudo docker exec -it ubuntu2 /bin/bash
\end{verbatim}

\begin{verbatim}
在容器内部执行 exit 命令退出时，该容器也随之停止。
\end{verbatim}
